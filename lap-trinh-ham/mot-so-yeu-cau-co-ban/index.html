Lập trình Hàm

  • Chương 1 - Một số yêu cầu cơ bản

Lập trình Hàm( Functional programming ) và Lập trình hướng đối tượng( object-oriented programming ) là 2 kiểu lập trình chính, phổ biến nhất trong thế giới lập trình.Trong nhiều năm qua cả 2 không nhìn mặt nhau,một số như C,python,javascript theo lập trình Hàm ,còn một số thì trung thành tuyệt đối với hướng đối tượng như C++,Java… Và rồi hôm nay, mọi chuyện thay đổi, rất nhiều ngôn ngữ lập trình coi trọng cả 2, Rust là 1 trong số đó.
Chương này chúng ta sẽ trình bày những nguyên tắc chung của lập trình hàm với 3 vấn đề sau:

  • Dùng lập trình hàm để đơn giản hóa, giảm khối lượng code
  • Viết code mạnh mẽ, an toàn với abstractions
  • Sử dụng lập trình hàm cho những dự án phức tạp

Dùng lập trình hàm để đơn giản hóa, giảm khối lượng code

Để generics càng generic hơn nữa

không biết phải dịch generic là gì nữa, các bạn xem lại cơ bản generic tại đây.
Nếu không có generic thì truct sẽ trở nên rườm ra (tốn nhiều code), ví dụ nhẹ sau :

struct PointU32 
{
    x: u32,
    y: u32
}
struct PointF32
{
    x: f32,
    y: f32
}
struct PointI32
{
    x: i32,
    y: i32
}

Một struct tọa độ 2D có x,y trong đó có 3 kiểu dữ liệu khác nhau là u32,f32 và i32.
Thay cho sự rườm rà này ta dùng generic

struct Point<T>
{
    x: T,
    y: T
}

Tiếp theo là hàm cũng vậy, có generic mọi thứ sẽ ngắn gọn hơn.
Code này là ko generic

fn foo_u32(x: u32) -> u32
{
    x*x
}
fn foo_f32(x: f32) -> f32
{
    x*x
}
fn foo_i32(x: i32) -> i32
{
    x*x
}
``
3 Hàm foo với 3 kiểu dữ liệu u32,f32,i32.
Chúng ta dùng generic 
```html
fn foo(x: T) -> T
where T: std::ops::Mul + Copy
{
    x*x
}

Tham số đầu vào cho hàm có thể là 1 hàm số khác

fn bar(f: F, x: T) -> T
where F: Fn(T) -> T
{
    f(x)
}

Hàm số như là giá trị ( functions as values )

Đại loại chúng ta đối xử với hàm như là 1 biến, đây là chìa khóa của tính đa hình(paradigm).Chúng ta sẽ nói về closure, 1 dạng của hàm implementing fn, Fn, FnMut, or FnOnce.

|Biến đầu vào| {
// code bên trong
}
Hoặc nếu ngắn gọn thì |Biến đầu vào| thực thi 1 dòng code
vd : |x| x*x

Nói chung khái y như như 1 hàm bình thường, ủa rồi cần closure làm gì,closure là hàm khai báo và dùng xong rồi thôi,kiểu 1 hàm không tên, mình thường dùng như là 1 biến đầu vào cho các hàm khác, ví dụ:


(0..10).map(|x| {
    fn f(y: u32) -> u32 {
        y*y
    }
    let z = f(x+1) * f(x+2);
    z*z
}
// Biến đầu vào hàm map là 1 closure
