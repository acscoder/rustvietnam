Một con trỏ (a pointer) là một biến được dùng để lưu trữ địa chỉ của biến khác.
Ở bài https://rustvietnam.com/lap-trinh-co-ban-rust/ownership-va-borrowing/ ta đã biết về tham chiếu biến với &.

let x = 10;
let y = &x;

Khác với tham chiếu, con trỏ là một biến có địa chỉ độc lập so với vùng nhớ mà nó trỏ đến, nhưng giá trị bên trong vùng nhớ của con trỏ chính là địa chỉ của biến (hoặc địa chỉ ảo) mà nó trỏ tới.

Con trỏ thông minh với Box

Biến con trỏ với Box thì dữ liệu sẽ được lưu trên bộ nhớ heap. Sau đây là khai báo biến với con trỏ Box

let five = Box::new(5);

Hoặc phức tạp hơn với 1 struct

struct Post {
    timestamp: i32,
    title: &'static str
} 
fn main() {
    let mut p1 = Box::new(Post {
        timestamp: 1829384,
        title: "The Box"
    });
    println!("{}", p1.title);
}

Như đã thấy , biến đầu vào của khởi tạo Box::new là bất kì dạng dữ liệu nào.Nhưng
với ví dụ vừa rồi chúng ta cũng không thể hiểu được Box thì khác gì với biến thường , khác gì với tham chiếu.Chúng ta thử ví dụ khác.

fn main() {
    let x = 5;
    let y = &x;
    let z = Box::new(x);
    println!("x {},y {},z {},",x,y,z);
}

Kết quả in ra là 5 như nhau.
Vậy ta thử so sánh biến bằng macro assert_eq!.

fn main() {
    let x = 5;
    let y = &x;
    let z = Box::new(x);
    println!("x {},y {},z {},",x,y,z);
    assert_eq!(x,y);
    assert_eq!(x,z);
    assert_eq!(y,z);
}

Chúng ta có kết quả là cả 3 so sánh đều không pass, không có biến nào giống nhau

error[E0277]: can't compare {integer} with &{integer}

error[E0277]: can't compare {integer} with Box<{integer}>

error[E0277]: can't compare &{integer} with Box<{integer}>

Vậy khác biệt đầu tiên ở dạng dữ liệu, kiểu bộ nhớ.Chúng ta thay đoạn so sánh trên bằng 1 so sánh với dereferencing biến bằng dấu sao *

assert_eq!(x,*z);
assert_eq!(*y,*z);

Test pass, biến z sau khi dereferencing giá trị sẽ bằng biến x , biến y cũng vậy. Tới lúc này mình thấy con trỏ Box z là 1 biến độc lập nhưng trỏ tới 1 vùng nhớ chứa giá trị của biến x , tạm thời thì chưa thấy sự khác biệt với tham chiếu y.
Một tham chiếu thì chỉ tham chiếu, không gì khác nhưng 1 con trỏ Box thì nhiều hơn thế rất nhiều.Một ví dụ về đệ qui

enum List {
    Cons(i32, List),
    Nil,
}
use crate::List::{Cons, Nil};
fn main() {
    let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
}


Đây là ý định của chúng ta , 1 định dạng lồng nhau liên tục (kiểu này người ta gọi là dạng dữ liệu đệ qui)
Nhưng code trên không thực hiện được với lỗi

error[E0072]: recursive type List has infinite size

Rust không định hình được kích thước của biến sẽ như thế nào vì việc lồng nhau sẽ ko xác định, nên định dạng này cần 1 con trỏ

enum List {
    Cons(i32, Box<List>),
    Nil,
}
use crate::List::{Cons, Nil};
fn main() {
    let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
}

Chỉ thay List bằng Box
