Tổng quan về lập trình hệ thống nhúng với Rust

Giới thiệu tổng quan về lập trình hệ thống nhúng với ngôn ngữ Rust.

Lập trình nhúng

  • Bài 1 - Tổng quan về lập trình hệ thống nhúng với Rust

Lập trình nhúng(embeded) được thực hiện với các bo mạch chuyên biệt có CPU,RAM thấp hoặc yêu cầu cao về tốc độ, lập trình trong nhúng hầu như yêu cầu code được biên dịch trực tiếp ra file thực thi
Trừ một số dự án gần đây người ta làm 1 bộ runtime cho Python https://micropython.org/ là có thể code với ngôn ngữ bậc cao , còn lại là đất diễn của C/C++ , với ưu điểm của riêng mình code Rust có thể được viết 1 cách dễ dàng như Python nhưng tốc độ thực thi chương trình ko kém cạnh gì C++ (thua C một ít nhé) ,ngoài ra còn bonus thêm khả năng quản lí lỗi, quản lí bộ nhớ , dễ dàng trong trong lập trình đa luồng, Rust thực sự là ngôn ngữ hứa hẹn cho các dự án nhúng trong thời gian tới, nhất là IoT.

Nói về lập trình nhúng(embeded) thì chúng ta phải nói về CPU hay gọi là Chíp nhúng/vi xử lí,vi điều kiển (làm phần mềm hay web thì chả ai quan tâm)
Trong mối quan hệ với Chíp nhúng chúng ta chia qua 3 tầng

1- Cấp thấp nhất - Assembly

Mỗi hãng , mỗi dòng Chíp có kiến trúc khác nhau , muốn lập trình chúng ta phải đọc datasheet để nắm cấu trúc, bộ nhớ,ngắt , các chân ra vào ...
Đây là 1 ví dụ datasheet của chip Pic32 (mình biết con này hồi đi học)
https://www.alldatasheet.com/view.jsp?Searchword=Pic32%20datasheet&gclid=Cj0KCQjwnJaKBhDgARIsAHmvz6cjGto75TORZ-Qb1-Fkob6hzCKQT6Wcb1KrB0J3fVy3WTJyLwVmwBgaAqdmEALw_wcB
Để điều khiển từng vùng nhớ, ngắt ... thì code Assembly là nền tảng nhất

nhưng dĩ nhiên Assembly quá rắc rối , vậy mới sinh ra C , C phù hợp với con người nó sẽ sản sinh Assembly - thứ phù hợp với máy --> code binary

Thay vì C chúng ta xài Rust, ở cấp độ này code phải nằm trong block unsafe , những đoạn code unsafe không phải là code dở,thiếu an toàn, mà code ở trong những vùng mà compile không thể kiểm tra trước được, cũng ko đảm bảo toàn bộ nhớ được, tự làm tự chịu

Ở tầng này Rust không sánh được với C.
Code trên là 1 ví dụ cho có thôi chứ ai làm thì làm, mình ko chơi code thấp kiểu đó đâu , ko vui chút nào.

SVD File và Peripheral Access Crate (PACs)

Ở cấp này nhà cung cấp chip cho chúng ta những SVD File, ví dụ:
https://www.keil.com/pack/doc/CMSIS/SVD/html/svd_Example_pg.html
Chúng ta có những thư viên(Crate) trong Rust sử dụng những file này để triển khai code dễ hơn
https://github.com/rust-embedded/svd2rust
Và chúng ta dùng thư viện Peripheral Access Crate (PACs) để code , tùy vào dòng chip mà ta có các Crate khác nhau.
Ví dụ crate lpc55-pac này,xài cho Chíp ARM Cortex-M33
https://crates.io/crates/lpc55-pac

Lúc này chúng ta ko còn cần unsafe nữa , các tính năng an toàn đã quay trở lại,zero code abstraction đã có, code cũng dễ dàng hơn , tất các tính năng trên chip được truy cập đầy đủ.Lúc này xài Rust được rồi nhưng thành thực mà nói mình cũng ngại(lười/ko có time) nên chưa thử.
Ở tầng này chúng ta cũng vẫn còn cần phải đọc datasheet để code, tuy nhiên Rust đã có thể so sánh được với C về khả năng viết chương trình, hiệu năng thì thế nào mình chưa biết.

Hardware Abstraction(HALs)

HALs là inteface cấp cao hơn trên nền của PACs
ví dụ
PACs cho ARM Cortex-M33 là
https://crates.io/crates/lpc55-pac
thì trên nền đó người ta cung cấp HALs
https://lib.rs/crates/lpc55-hal

Code dễ dàng hơn bao giờ hết, tầng này Rust có thể so sánh được với C++ về khả năng viết chương trình lẫn hiệu năng code.
HALs có interface để dễ dàng code giao tiếp thiết bị ngoại vi như cảm biến,bàn phím...
Ví dụ như cái cảm biên siêu âm đo khoảng cách này

Dĩ nhiên các HALs phụ thuộc vào các nhà cung cấp, không phải Chip hay thiết bị ngoại vi nào cũng được cung cấp ,Rust còn mới nhất là với lập trình nhúng.
Ưu thế của Rust phát huy đầy đủ ở tầng này, hầu như các code lập trình nhúng với Rust sẽ ở đây.

Board Support Crate (BSP)

Tưởng chia 3 tầng là xong mà còn có thằng này nữa.
Board Support Crate (BSP) là các thư viện được viết sẵn cho các board mạch, cấp này dễ dàng hơn Hardware Abstraction(HALs) vì HALs cho Chip mà thôi, xếp nó cũng ở tầng 3 nhưng cao hơn HALs.
Ví dụ thư viện này https://github.com/esp-rs/rust cho 2 bo mạch ESP32 và ESP8266 , 2 con này thực sự ngon, rẻ và cực phổ biến trong IoT.

Lập trình thời gian thực

Lập trình đa luồng vẫn thực hiện tốt với Chip đơn 1 core bằng các ngắt , người ta gọi là Interrupt-driven , hay Lập trình thời gian thực , ko biết mình có hiểu lầm không nữa nhưng mình thấy Lập trình thời gian thực và Interrupt-driven là như nhau.
freeRTOS là một hệ điều hành nhúng thời gian thực được viết bằng ngôn ngữ C và rất phổ biến trên Arduino
http://arduino.vn/bai-viet/1673-chay-da-nhiem-tren-arduino-voi-freertos - Tham khảo cho ai chưa biết.
Giống như freeRTOS của C chúng ta cũng có những Interrupt-driven mạnh mẽ với Rust
https://rtic.rs/0.5/book/en/
Giúp thực hiện code đa luồng và thời gian thực.
