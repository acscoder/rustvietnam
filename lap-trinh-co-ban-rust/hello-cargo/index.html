Cargo là hệ thống quản lí thư viện cho Rust, như npm trong nodejs hay composer trong PHP
Thường thì chúng ta sẽ tạo 1 project trong Cargo chứ ít khi viết rồi compile bằng rustc như ở bài trước
Cargo đã được cài lúc bắt đầu, chúng ta kiểm tra version

cargo --version

Nếu chúng ta thấy số phiên bản, ok good,không thì phải quay lại phần cài đặt
Cargo cung cấp cho chúng ta phương tiện làm việc tốt hơn, tạo 1 project mới bẳng lệnh cargo new

cargo new hello_cargo
cd hello_cargo

Kiểm tra thư mục chúng ta sẽ thấy 1 folder mới tên hello_cargo được tạo vào trong chúng ta sẽ thấy file Cargo.toml - đây là file khai báo thông tin project của chúng ta và các module thư viện( crate ) các thư viện trong Rust gọi là crate , như trong Nodejs gọi là package
Mở ta sẽ thấy

[package]
name = "hello_cargo"
version = "0.1.0"
authors = ["Your Name you@example.com"]
edition = "2018"
[dependencies]
// các crate sẽ được khai báo ở đây

File tiếp theo được tạo ra là src/main.rs đây là file code chính của project, trong file có sẵn hàm main() chờ chúng ta bắt đầu.

fn main() {
println!("Hello, world!");
}

Vậy là chúng ta cũng có 1 Hello, world! nữa
với Cargo , chỉ khác lần này tự hệ thống tạo nó , cấu trúc thư mục khác 1 chút, tất cả code nằm trong folder src , và khi compile code mình dùng câu lệnh khác

cargo build

Lệnh này compile code của chúng ta vào thư mục ./target/debug/ hoặc ./target/release nếu chúng ta release sản phẩm cuối cùng (cargo build --release)

./target/debug/hello_cargo
//Ta sẽ có Hello, world!

1 cách khác gọn hơn, ta chạy lệnh

cargo run

1 câu lệnh khác là cargo check nếu mình chỉ muốn kiểm tra xem code mình có lỗi gì không rồi code tiếp chứ chưa muốn compile.
