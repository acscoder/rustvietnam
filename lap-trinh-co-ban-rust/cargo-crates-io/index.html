Tùy biến cargo build với option release

Cargo có 2 profile chính 1 là dev - bản đang phát triển, compile ra để kiểm tra,debug
2 là bản release xuất ra sản phẩm, mặc định là dev

$ cargo build
    Finished dev [unoptimized + debuginfo] target(s) in 0.0s
$ cargo build --release
    Finished release [optimized] target(s) in 0.0s

ta thêm và --release , bản release sẽ được optimized (tối ưu code)
Trong file config Cargo , chúng ta có thể khai báo riêng cho dev,release

[profile.dev]
opt-level = 0

[profile.release]
opt-level = 3

Ở trên mình set cấp độ tối ưu hóa code opt-level, cấp độ cao thì tối ưu nhiều, tốn thời gian nhiều

crates.io

crates.io là site quản lí tất cả các thư viện của Rust.Chúng ta có thể lên đó tìm crate mình cần hoặc chia sẽ các crates mình viết.
Để publish crate mình viết thì trước hết code mình phải comment document đầy đủ, xem lại
Sau đó, tạo 1 account trong crates.io lấy API token tại https://crates.io/me/
Sau đó chạy lệnh cargo login kiểu như vầy

cargo login abcdefghijklmnopqrstuvwxyz012345

Token sẽ được lưu trong ~/.cargo/credentials
Điền đầy đủ các thông tin crate như name,version ,description, license trong Cargo.toml

[package]
name = "guessing_game"
version = "0.1.0"
edition = "2018"
description = "A fun game where you guess what number the computer has chosen."
license = "MIT OR Apache-2.0"
[dependencies]

Chạy lệnh publish

cargo publish

Bin và Lib

Với lệnh cargo new chúng ta tạo 1 project với file thực thi main.rs , project sau khi compile sẽ tạo 1 file thực thi .exe(windown) để chạy chương trình

 ├── examples_bin
 │  ├── Cargo.toml
 │     └── src
 │        └── main.rs

Vẫn còn 1 kiểu project là phát triển 1 thư viện cho các project sử dụng

cargo new examples_lib --lib

 ├── examples_lib
 │  ├── Cargo.toml
 │     └── src
 │        └── lib.rs

Khác nhau ở file main.rs và lib.rs
Vậy 1 project vừa có thể compile để thực thi và compile để tạo thư viện có được không.
Được , chúng ta tạo cả 2 file main.rs và lib.rs luôn

 ├── examples_lib
 │  ├── Cargo.toml
 │     └── src
 │        └── main.rs 
          └── lib.rs

Sau đó khai báo đường dẫn trong Cargo.toml

[lib]
name = "mylib"
path = "src/lib.rs"
[[bin]]
name = "mybin"
path = "src/bin.rs"

Mình sử dụng thư viện lib trong main.rs như sau

//Tên thư viện:: hàm,biến cần dùng
use mylib::test;
pub fn main() {
    test();
}

Thư mục bin

Một cách khác để tách riêng main.rs và lib.rs là bỏ main.rs đi tạo 1 folder bin , trong folder bin ta tạo các file .rs để biên dịch

├── Cargo.toml
└── src
    ├── bin
    │   └── mybin.rs
    └── lib.rs

Code ví dụ trong lib.rs

use std::error::Error;

pub fn really_complicated_code(a: u8, b: u8) -> Result<u8, Box<dyn Error>> {
    Ok(a + b)
}

Ta sử dụng lib trong bin/mybin.rs

fn main() {
    println!(
        "I'm using the library: {:?}",
        example_project::really_complicated_code(1, 2)
    );
}

Lưu ý example_project là tên project mình đã khai báo trong Cargo.toml

[package]
name = "example_project"
version = "0.1.0"
edition = "2018"

Để biên dịch mybin.rs ta chạy lệnh

cargo run --bin mybin -q

Workspaces

Nhiều project trong một không gian chung - Workspaces
Ví dụ , chúng ta tạo 1 dự án

cargo new examples_workspace

Sau đó chạy

cargo new examples_workspace/lib --lib

cargo new examples_workspace/one_more

 ├── examples_workspace
 │  ├── Cargo.toml
 │     └── src
 │        └── main.rs  
       └── lib
          |── Cargo.toml
 │        └── src
 │          └── lib.rs  
 │    ├── one_more
 │        ├── Cargo.toml
 │          └── src
 │             └── lib.rs

Trong examples_workspace/Cargo.toml chúng ta khai báo lib và one_more bên trong workspace

[workspace]
members = [
"lib",
"one_more"
]
