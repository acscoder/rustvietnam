Enum

Enum (enumerate : liệt kê,danh sách) như tên gọi, đây là kiểu dữ liệu liệt kê danh sách, chỉ có tên chứ không có giá trị , chuyên dùng để so sánh

enum IpAddrKind {
    V4,
    V6,
}

Ví dụ trên ta định nghĩa 1 enum IpAddrKind - các loại địa chỉ Ip, thì ta có 2 loại là V4 và V6 trong danh sách
Sử dụng enum bằng dấu :: , ví dụ

let four = IpAddrKind::V4;
let six = IpAddrKind::V6;

Ví dụ sử dụng enum đơn giản như sau

enum IpAddrKind {
        V4,
        V6,
    }
fn main()
{
    let home = (IpAddrKind::V4,"192.168.1.1");
    let work = (IpAddrKind::V6,"::1");
    check_ip(home);
    check_ip(work);
}
fn check_ip(ip:(IpAddrKind,&str)){
    match ip.0 {
        IpAddrKind::V4 => println!("Địa chỉ {} là V4",ip.1),
        IpAddrKind::V6 => println!("Địa chỉ {} là V6",ip.1),
    }
}

Hàm check_ip dùng so sánh match để nhân ra V4 hay V6
Enum không chỉ có vậy, chúng ta có thể khai báo kết hợp với các kiểu dữ liệu cơ bản để tạo ra kiểu dữ liệu phức tạp hơn.

enum IpAddr {
        V4(String),
        V6(String),
    }
let home = IpAddr::V4(String::from("127.0.0.1"));
let loopback = IpAddr::V6(String::from("::1"));

Vậy kiểu dữ liệu chúng ta không cố định IpAddr::V4 mà là IpAddr::V4("bất kì string bên trong")

Struct

Struct là một kiểu dữ liệu dạng list do ta tự định nghĩa, Struct kết hợp với trait được sử dụng giống như class trong ngôn ngữ hướng đối tượng, ta xem 1 vài các khai báo

#[derive(Debug)]
struct Person {
    name: String,
    age: u8,
}
fn main(){
    let author = Person{name:"Ann Lee".to_string(),age:35};
    println!("{:?}",author);
}

Khai báo Struct dạng tuple

struct Pair(i32, f32);

Khai báo Struct rỗng

struct Unit;

Trait

Trait như là 1 interface trong hướng đối tượng nhưng có nhiều khác biệt(dĩ nhiên phần cơ bản thì ko thể trình bày hết được).
Trait được gọi là các hàm có thể chia sẽ cho các dạng dữ liệu
Ví dụ chúng ta khai báo 2 dạng dữ liệu struct là NewsArticle và Tweet ,1 trait Summary

pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}
pub struct Tweet {
    pub username: String,
    pub content: String,
    pub reply: bool,
    pub retweet: bool,
}
pub trait Summary {
    fn summarize(&self) -> String;
    fn readmore_method(&self) -> String {
        String::from("(Read more...)")
    }
}

Tiếp theo cả 2 NewsArticle và Tweet đều chia sẽ (implement) hàm của trait Summary , ta dùng keyword impl .. for

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}
impl Summary for Tweet {
    fn summarize(&self) -> String {
        format!("{}: {}", self.username, self.content)
    }
}

Sử dụng NewsArticle và Tweet

fn main(){ 
let article = NewsArticle {
        headline: String::from("Penguins win the Stanley Cup Championship!"),
        location: String::from("Pittsburgh, PA, USA"),
        author: String::from("Iceburgh"),
        content: String::from(
            "The Pittsburgh Penguins once again are the best \
             hockey team in the NHL.",
        ),
    };

    println!("New article available! {}", article.summarize());
println!("{}", article.readmore_method());

let tweet = Tweet {
        username: String::from("horse_ebooks"),
        content: String::from(
            "of course, as you probably already know, people",
        ),
        reply: false,
        retweet: false,
    };
    println!("1 new tweet: {}", tweet.summarize());
    println!("{}", tweet.readmore_method());
}

Tiếp theo chúng ta có hàm

    println!("Breaking news! {}", item.summarize());
}

Hàm này nhận dạng dữ liệu mà đã impl trail
Summary vô rồi fn notify
Ta sử dụng hàm notify này bên trong main() thêm 2 dòng

notify(&article);
notify(&tweet);

ok , chúng ta sẽ thấy biến đầu vào không phải là notify(article) mà notify(&article), gọi là tham chiếu biến , bài sau về Ownership và Borrowing chúng ta sẽ hiểu rõ hơn nhé.
