Trong file src/main.rs , hoặc mở https://play.rust-lang.org/

fn main() {
    let x = 5;
    println!("The value of x is: {}", x);
    x = 6;
    println!("The value of x is: {}", x);
}

1 đoạn code nhỏ biến x được khái báo với keyword let . Chạy đoạn code ta nhận ngay 1 lỗi

Mặc định khi khai báo biến thì biến là 1 immutable, giá trị biến là cố định ko thay đổi thêm nữa

cannot assign twice to immutable variable x

Để có thể thay đổi giá trị thì mình phải khai báo biến là một mutable với keyword mut

let mut x = 5;

Thay let x = 5; bằng let mut x = 5; chương trình chúng ta sẽ chạy mà ko có lỗi gì.
Vậy chúng ta coi biến immutable như let x = 5; là một hằng số có được không.
Câu trả lời là không, ta gọi nó là biến immutable chứ ko gọi nó là hằng số (constant)
Vậy điểm khác biệt là gì.
Một biến immutable tuy không đổi nhưng nó có thể được khai báo lại trong quá trình dùng, ví dụ

fn main() {
    let x = 5;
    println!("The value of x is: {}", x);
    let x = 6;
    println!("The value of x is: {}", x);
}

Chúng ta lặp lại khai báo biến let x = 6; thì giá trị của x thay đổi , cái này gọi là Shadowing , chuyện này ko xảy ra với hằng số, hằng số là cố định ko khai báo lại hay khai báo mut như biến được

const MAX_POINTS: u32 = 100_000;

Lưu ý khi Shadowing thì biến sau phải cùng dạng dữ liệu với ban đầu nếu không code sẽ lỗi. Ví dụ như vầy là lỗi

fn main() {
    let spaces = "   ";
    let spaces = spaces.len();
}

Ban đầu khai báo dạng dữ liệu là chuỗi nhưng tiếp theo đổi thành số thì sẽ không được chấp nhận.
Chúng ta sẽ nói thêm về dạng dữ liệu (data types) ở bài sau.
