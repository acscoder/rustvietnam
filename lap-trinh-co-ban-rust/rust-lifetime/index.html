Chúng ta bắt đầu bằng 1 code so sánh độ dài 2 chuỗi

fn main() {
let string1 = String::from("abcd");
let string2 = "xyz";
let result = longest(string1.as_str(), string2);
println!("The longest string is {}", result);
}
fn longest(x: &str, y: &str) -> &str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

Đoạn code đơn giản trên báo lỗi

error[E0106]: missing lifetime specifier

Để hiểu vấn đề này, chúng ta nói vê lifetime, chúng ta đã biết biến trong Rust có quyền sỡ hữu biến và được giải phóng sau khi dùng xong , mỗi biến có lifetime - thời gian tồn tại của riêng mình

fn main() {
    {
        let r;                // ---------+-- 'a
                              //          |
        {                     //          |
            let x = 5;        // -+-- 'b  |
            r = &x;           //  |       |
        }                     // -+       |
                              //          |
        println!("r: {}", r); //          |
    }                         // ---------+
}

Ở ví dụ trên biến x có lifetime - "đời sống" kí hiệu 'a , tồn tại từ đầu đến cuối main , biến x lifetime - "đời sống" kí hiệu 'b tồn tại trong dấu { } nhỏ hơn , chúng ta thấy biến r tham chiếu từ x r = &x;
sau khi ra khỏi lifetime 'b kết thúc, dẫn tới việc tham chiếu bị mất hiệu lực, println!("r: {}", r); lỗi.
Quay lại với ví dụ đầu tiên chúng ta xét hàm longest , hàm này trả về biến x hoặc y tức là

fn longest(x: &str, y: &str) -> &str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

Vậy là biến result sẽ được gán string1 hoặc string2 và lifetime trả về là của biến nào cũng chưa rõ dẫn đến báo lỗi

let result = longest(string1.as_str(), string2);

Giờ chúng ta phải thêm 1 nhiệm vụ là khai báo lifetime , lifetime kí hiệu là 'tên_lifetime, hàm longest sẽ viết lại là

fn main() {
    let string1 = String::from("abcd");
    let string2 = "xyz";
    let result = longest(string1.as_str(), string2);
    println!("The longest string is {}", result);
}
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

2 đầu vào dùng chung 1 lifetime là 'a , trả về cũng là 'a , dù biến trả về là biến nào thì lifetime là 'a và nó vẫn có hiệu lực khi mình dùng code println!("The longest string is {}", result); --> work
Có thể bạn nghĩ việc kiểm soát "đời sống" của biến là cái gì đó có vẻ kì quặc,mình cũng đồng ý , khi tiếp tục với những code phức tạp hơn mình sẽ đối mặt với những vấn đề lifetime phức tạp hơn nhưng cái giá này tụi mình trả để có được những đoạn code an toàn, chạy nhanh và ổn định, thực sự thì ko tệ đâu.
